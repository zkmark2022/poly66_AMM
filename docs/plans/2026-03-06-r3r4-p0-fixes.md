# R3R4 P0 Fixes Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Fix the three P0 issues in the AMM bot so the production entrypoint uses the full Oracle implementation, the A-S engine produces sane spreads around mid=50, and `AMMReconciler` no longer crashes startup.

**Architecture:** Keep a single Oracle implementation in `src/amm/oracle/polymarket_oracle.py`, convert its fetch/refresh path to async, and wire `src/amm/main.py` to that implementation only. Adjust A-S time scaling at the pricing formula boundary rather than layering more clamps, so quotes remain in a realistic cents range without breaking inventory skew semantics. Fix the reconciler startup call at the composition root and cover it with a regression test.

**Tech Stack:** Python 3, pytest, pyright, ruff, asyncio, unittest.mock
